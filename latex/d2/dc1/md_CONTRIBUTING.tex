\doxysection*{How we work}

Our workflow is based on {\bfseries{Git Flow}}, outlined in the steps below. ~\newline
 As always, please fix only one issue at a time. ~\newline
 Build your branch with Qt and test, then make a pull request. ~\newline


Read more about {\bfseries{Git Flow}}\+: \href{http://nvie.com/posts/a-successful-git-branching-model/}{\texttt{ http\+://nvie.\+com/posts/a-\/successful-\/git-\/branching-\/model/}} ~\newline
 Read more about {\bfseries{Pull Requests}}\+: \href{https://help.github.com/articles/about-pull-requests/}{\texttt{ https\+://help.\+github.\+com/articles/about-\/pull-\/requests/}} and \href{https://help.github.com/articles/creating-a-pull-request/}{\texttt{ https\+://help.\+github.\+com/articles/creating-\/a-\/pull-\/request/}} ~\newline


\doxysection*{Naming conventions}

Name your branch after the issue you will fix. Example\+: Name your branch {\bfseries{issue-\/\#155}} for issue \#155. The permanent named branches on this repo are\+:
\begin{DoxyItemize}
\item {\bfseries{develop}} -\/ Contains next major release. Used for testing and sharing among developers.
\item {\bfseries{master}} -\/ Used only for releases.
\end{DoxyItemize}

\doxysection*{Clone the repo \& create your issue branch}


\begin{DoxyItemize}
\item Clone the repo, set the origin URL, switch focus to develop branch, and create your feature branch. Examples use branch for issue \#155. 
\begin{DoxyCode}{0}
\DoxyCodeLine{git clone https://github.com/fashionfreedom/seamly2d  }
\DoxyCodeLine{git remote set-\/url origin https://github.com/fashionfreedom/seamly2d}
\DoxyCodeLine{git checkout develop       }
\DoxyCodeLine{git checkout -\/b issue-\/\#155   }
\DoxyCodeLine{git checkout issue-\/\#155    }

\end{DoxyCode}

\end{DoxyItemize}

\doxysection*{Daily workflow}


\begin{DoxyItemize}
\item Pull latest changes from the origin {\bfseries{develop}} branch \& merge them into your issue branch 
\begin{DoxyCode}{0}
\DoxyCodeLine{git pull origin develop  }
\DoxyCodeLine{git checkout issue-\/\#155  }
\DoxyCodeLine{git merge develop}

\item Make your code changes \& commit your changes. ~\newline
 
\begin{DoxyCode}{0}

\end{DoxyCode}

\end{DoxyItemize}

\doxysection*{Create a Pull Request -- git CLI}


\begin{DoxyItemize}
\item Build \& test your issue branch with Qt 5.\+15.\+2 or Qt 5.\+15.\+12, and Qt Creator
\item Push your issue branch up to the github repo 
\begin{DoxyCode}{0}
\DoxyCodeLine{git push -\/u origin issue-\/\#155}

\end{DoxyCode}

\item In the main repo on github, on the main menu bar, select {\bfseries{Branch\+: develop}} then {\bfseries{Pull Requests}}
\item From the {\bfseries{Compare\+:develop}} dropdown list, select the green {\bfseries{New Pull Request}} button then select your issue branch
\item Enter the Pull Request Header as $\ast$$\ast$\char`\"{}$<$tag$>$\+: $<$description issue\+\_\+\#nnn$>$\char`\"{}$\ast$$\ast$ using $\ast$$\ast$\+\_\+conventional-\/changelog/commit-\/zen-\/cli\+\_\+$\ast$$\ast$ tags so your commits are auto-\/included in the Changelog.\+md file. Read more here\+: \href{https://docs.changelogg.io/conventional-changelog-template}{\texttt{ https\+://docs.\+changelogg.\+io/conventional-\/changelog-\/template}}. Example PR Header\+: \char`\"{}fix\+: Arc \& line tool \#issue-\/\#155\char`\"{}. ~\newline

\begin{DoxyItemize}
\item Valid Tags are\+:
\begin{DoxyItemize}
\item fix
\item feat
\item build
\item ci
\item chore
\item docs
\item perf
\item refactor
\item revert
\item style
\item test
\end{DoxyItemize}
\end{DoxyItemize}
\item Click the {\bfseries{Create Pull Request}} button.
\item When prompted, sign the {\bfseries{Code Licensing Agreement (CLA)}}. You sign this only once.
\item Select {\bfseries{Linked Issues}} in the right hand column and select the issue. Example\+: select issue \#155. ~\newline

\item The maintainer will check that all CI/\+CD tests passed, review the code, then merge your issue branch into {\bfseries{develop}}.
\item Releases are weekly on Monday morning 01\+:30am UTC.
\end{DoxyItemize}
