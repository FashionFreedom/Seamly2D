33 /**
34  * This class is a derivative of the standard InputSource class. It provides
35  * for the parser access to data stored in a memory buffer. The type of
36  * buffer and its host specific attributes are of little concern here. The
37  * only real requirement is that the memory be readable by the current
38  * process.
39  *
40  * Note that the memory buffer size is expressed in <b>bytes</b>, not in
41  * characters. If you pass it text data, you must account for the bytes
42  * per character when indicating the buffer size.
43  *
44  * As with all InputSource derivatives. The primary objective of an input
45  * source is to create an input stream via which the parser can spool in
46  * data from the referenced source. In this case, there are two options
47  * available.
48  *
49  * The passed buffer can be adopted or merely referenced. If it is adopted,
50  * then it must be dynamically allocated and will be destroyed when the
51  * input source is destroyed (no reference counting!.) Note that the
52  * deallocation assumes that array deletion should be performed, so do
53  * not pass a non-array-allocated buffer if asking for adoption.
54  * If not adopted, the caller must insure that it remains valid until the
55  * input source object is destroyed.
56  *
57  * The other option indicates whether each stream created for this input
58  * source should get its own copy of the data, or whether it should just
59  * stream the data directly from this object's copy of the data. The same
60  * rules apply here, in that the buffer must either be copied by the
61  * stream or it must remain valid until the stream is destroyed.
62  */

73  /**
74  * A memory buffer input source is constructed from a buffer of byte
75  * data, and the count of bytes in that buffer. The parser will parse
76  * from this memory buffer until it has eaten the indicated number of
77  * bytes.
78  *
79  * Note that the system id provided serves two purposes. Firstly it is
80  * going to be displayed in error messages as the source of the error.
81  * And secondly, any entities which are referred to from this entity
82  * via relative paths/URLs will be relative to this fake system id.
83  *
84  * @param srcDocBytes The actual data buffer to be parsed from.
85  * @param byteCount The count of bytes (not characters, bytes!)
86  * in the buffer.
87  * @param bufId A fake system id for the buffer.
88  * @param adoptBuffer Indicates whether this object should adopt

158  /**
159  * By default, for safety's sake, each newly created stream from this
160  * input source will make its own copy of the buffer to stream from. This
161  * avoids having to deal with aliasing of the buffer for simple work. But,
162  * for higher performance applications or for large buffers, this is
163  * obviously not optimal.
164  *
165  * In such cases, you can call this method to turn off that default
166  * action. Once turned off, the streams will just get a pointer to the
167  * buffer and parse directly from that. In this case, you must insure that
168  * the buffer remains valid for as long as any parse events are still
169  * using it.
170  *
