28 /**
29  * User Documentation for Xerces Version Values:
30  *
31  *
32  *
33  * Xerces Notes:
34  *
35  * Xerces Committers Documentation:
36  *
37  * Xerces committers normally only need to modify one or two of the
38  * following macros:
39  *
40  * XERCES_VERSION_MAJOR
41  * XERCES_VERSION_MINOR
42  * XERCES_VERSION_REVISION
43  *
44  * The integer values of these macros define the Xerces version number. All
45  * other constants and preprocessor macros are automatically generated from
46  * these three definitions.
47  *
48  * The macro XERCES_GRAMMAR_SERIALIZATION_LEVEL has been added so that during
49  * development if users are using the latest code they can use the grammar
50  * serialization/deserialization features. Whenever a change is made to the
51  * serialization code this macro should be incremented.
52  *
53  * Xerces User Documentation:
54  *
55  * The following sections in the user documentation have examples based upon
56  * the following three version input values:
57  *
58  * #define XERCES_VERSION_MAJOR 19
59  * #define XERCES_VERSION_MINOR 3
60  * #define XERCES_VERSION_REVISION 74
61  *
62  * The minor and revision (patch level) numbers have two digits of resolution
63  * which means that '3' becomes '03' in this example. This policy guarantees
64  * that when using preprocessor macros, version 19.3.74 will be greater than
65  * version 1.94.74 since the first will expand to 190374 and the second to
66  * 19474.
67  *
68  * Preprocessor Macros:
69  *
70  * _XERCES_VERSION defines the primary preprocessor macro that users will
71  * introduce into their code to perform conditional compilation where the
72  * version of Xerces is detected in order to enable or disable version
73  * specific capabilities. The value of _XERCES_VERSION for the above example
74  * will be 190374. To use it a user would perform an operation such as the
75  * following:
76  *
77  * #if _XERCES_VERSION >= 190374
78  * // code specific to new version of Xerces...
79  * #else
80  * // old code here...
81  * #endif
82  *
83  * XERCES_FULLVERSIONSTR is a preprocessor macro that expands to a string
84  * constant whose value, for the above example, will be "19_3_74".
85  *
86  * XERCES_FULLVERSIONDOT is a preprocessor macro that expands to a string
87  * constant whose value, for the above example, will be "19.3.74".
88  *
89  * XERCES_VERSIONSTR is a preprocessor macro that expands to a string
90  * constant whose value, for the above example, will be "19_3". This
91  * particular macro is very dangerous if it were to be used for comparing
92  * version numbers since ordering will not be guaranteed.
93  *
94  * Xerces_DLLVersionStr is a preprocessor macro that expands to a string
95  * constant whose value, for the above example, will be "19_3_74". This
96  * macro is provided for backwards compatibility to pre-1.7 versions of
97  * Xerces.
98  *
99  * String Constants:
100  *
101  * gXercesVersionStr is a global string constant whose value corresponds to
102  * the value "19_3" for the above example.
103  *
104  * gXercesFullVersionStr is a global string constant whose value corresponds
105  * to the value "19_3_74" for the above example.
106  *
107  * Numeric Constants:
108  *
109  * gXercesMajVersion is a global integer constant whose value corresponds to
110  * the major version number. For the above example its value will be 19.
111  *
112  * gXercesMinVersion is a global integer constant whose value corresponds to
113  * the minor version number. For the above example its value will be 3.
114  *
115  * gXercesRevision is a global integer constant whose value corresponds to
116  * the revision (patch) version number. For the above example its value will
117  * be 74.
118  *
119  */
