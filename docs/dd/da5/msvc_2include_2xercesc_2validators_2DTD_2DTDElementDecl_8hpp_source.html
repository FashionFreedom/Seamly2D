38 // This class is a derivative of the basic element decl. This one implements
39 // the virtuals so that they work for a DTD. The big difference is that
40 // they don't live in any URL in the DTD. The names are just stored as full
41 // QNames, so they are not split out and element decls don't live within
42 // URL namespaces or anything like that.

144  // -----------------------------------------------------------------------
145  // Private data members
146  //
147  // fAttDefs
148  // The list of attributes that are defined for this element. Each
149  // element is its own little 'namespace' for attributes, so each
150  // element maintains its own list of owned attribute defs. It is
151  // faulted in when an attribute is actually added.
152  //
153  // fAttList
154  // We have to return a view of our att defs via the abstract view
155  // that the scanner understands. It may or may not ever be asked
156  // for so we fault it in as needed.
157  //
158  // fContentSpec
159  // This is the content spec for the node. It contains the original
160  // content spec that was read from the DTD, as a tree of nodes. This
161  // one is always set up, and is used to build the fContentModel
162  // version if we are validating.
163  //
164  // fModelType
165  // The content model type of this element. This tells us what kind
166  // of content model to create.
167  //
168  // fContentModel
169  // The content model object for this element. It is stored here via
170  // its abstract interface.
171  //
172  // fFormattedModel
173  // This is a faulted in member. When the outside world asks for
174  // our content model as a string, we format it and fault it into
175  // this field (to avoid doing the formatted over and over.)
176  // -----------------------------------------------------------------------
